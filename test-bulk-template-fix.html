Issue Summary

Only the first slideshow was getting text overlays applied during bulk template creation.

Root Cause

Text overlays kept their original slide indices from the template, but each slideshow group needed to have text overlays adapted to its specific image positions.

Fix Applied

  • Slide Index Adaptation: Text overlays are now properly mapped to slide indices within each slideshow group
  • Bounds Checking: If a template has more slides than images in a group, indices are capped appropriately
  • Debug Logging: Added comprehensive logging to track text overlay application

Code Changes

Before (Broken)

const adaptedTextOverlays = template.textOverlays.map(overlay => ({
    ...overlay,
    id: `${overlay.id}_${Date.now()}_${Math.random().toString(36).substr(2, 9)}`,
    slideIndex: overlay.slideIndex // ‚ùå Keep original slide index
}));

After (Fixed)

const adaptedTextOverlays = template.textOverlays.map(overlay => ({
    ...overlay,
    id: `${overlay.id}_${Date.now()}_${Math.random().toString(36).substr(2, 9)}`,
    slideIndex: overlay.slideIndex >= imageGroup.length ? imageGroup.length - 1 : overlay.slideIndex // ‚úÖ Adapt to current group
}));

Expected Behavior After Fix

Template: 4 slides with text overlays

  • Slide 1: "Welcome" text
  • Slide 2: "Title" text
  • Slide 3: "Subtitle" text
  • Slide 4: "Footer" text

Bulk Creation: 10 images, 4-slide template

  • Slideshow 1: Images 1-4 ‚Üí All 4 text overlays applied
  • Slideshow 2: Images 5-8 ‚Üí All 4 text overlays applied
  • Slideshow 3: Images 9-10 ‚Üí Text overlays for slides 1-2 applied (indices capped to available slides)

How to Test

  1. Create a template with multiple text overlays on different slides
  2. Upload more images than the template slide count
  3. Use "Create Multiple Slideshows from Template"
  4. Check that all slideshows have text overlays applied
  5. Check browser console for debug logs showing text overlay adaptation
